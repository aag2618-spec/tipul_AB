## 🚨 משימות ידניות דחופות לפני production deploy

### 1. **Prisma Migration** (קריטי!)

הוספנו 3 שדות nullable ל-User model:
- `passwordChangedAt DateTime?` (C7)
- `failedLoginAttempts Int @default(0)` (H11)
- `lockedUntil DateTime?` (H11)

לפני production deploy חייבים ליצור migration:

```powershell
cd "c:/Users/User/Documents/tipul_AB/tipul_AB-main"
npx prisma migrate dev --name security_2026_05_11 --create-only
# בדקי את ה-SQL ב-prisma/migrations/...
# אז ב-Render Shell:
npx prisma migrate deploy
```

⚠️ אם משתמש `prisma db push --accept-data-loss` ב-production (כפי שמתועד ב-`אבטחה-משימות-ידניות-להמשך.md`), השדות יתווספו אוטומטית בלי data loss (כולם nullable או default value).

### 2. **Environment Variables ב-Render**

```bash
WEBHOOK_REQUIRE_TIMESTAMP=true   # מפעיל strict mode ב-webhook replay protection
```
