# מדריך הקמה — שמירת קבצים בענן (Cloudflare R2)

## למה צריך את זה (בקצרה)

היום הקבצים (מסמכים, הקלטות, צרופות) נשמרים על "דיסק זמני" של השרת, שמתרוקן
בכל פעם שמעדכנים את האתר. לכן קבצים ישנים נעלמים ומופיע **"File not found"**.

הפתרון: לשמור את הקבצים ב**ענן מוצפן של Cloudflare (R2)**. שם הם נשמרים לתמיד,
מגובים ומוצפנים. **הקוד כבר מוכן** — צריך רק לפתוח חשבון R2 ולחבר אותו.

- ⏱️ זמן: ~10 דקות, פעם אחת.
- 💰 עלות: עד 10GB — **בחינם**. אחר כך כמה אגורות לחודש.

---

## חלק 1 — פתיחת "דלי" (Bucket) ב-Cloudflare

1. כנס/י ל-https://dash.cloudflare.com ופתח/י חשבון חינמי (או התחבר/י אם כבר יש).
2. בתפריט בצד לחץ/י על **R2 Object Storage** (או פשוט **R2**).
3. ייתכן שיבקשו פרטי תשלום (כרטיס אשראי) כדי להפעיל את R2 — זה תקין, החיוב יהיה
   ~0 בשימוש קטן.
4. לחץ/י **Create bucket** (יצירת דלי).
5. שם הדלי: `mytipul-uploads` (אפשר כל שם, **באנגלית בלבד**, בלי רווחים).
6. אם שואלים על מיקום (Location): בחר/י **Europe (EU)** — טוב לפרטיות, הנתונים
   נשארים באירופה.
7. לחץ/י **Create bucket**.

## חלק 2 — יצירת מפתחות גישה (API Token)

8. בעמוד R2, למעלה, לחץ/י על **Manage R2 API Tokens** (או "API" ← "Manage API
   Tokens").
9. לחץ/י **Create API Token** (או **Create Account API Token**).
10. שם הטוקן: `mytipul`.
11. הרשאות (Permissions): בחר/י **Object Read & Write** (קריאה וכתיבה לקבצים).
12. אם שואלים על איזה דלי (Specify buckets): בחר/י את `mytipul-uploads`
    (או "Apply to all buckets").
13. לחץ/י **Create API Token**.
14. עכשיו יוצגו ערכים — **העתק/י ושמור/י אותם מיד** (ה-Secret מוצג רק פעם אחת!):
    - **Access Key ID** — מחרוזת קצרה.
    - **Secret Access Key** — מחרוזת ארוכה.
    - **Endpoint** (כתובת ה-S3 API) — נראה כמו
      `https://xxxxxxxxxxxx.r2.cloudflarestorage.com`.

## חלק 3 — חיבור ל-Render (השרת של האתר)

15. כנס/י ל-https://dashboard.render.com ← השירות **tipul-app** ← לשונית
    **Environment**.
16. הוסף/י את 5 המשתנים הבאים (כפתור **Add Environment Variable** לכל שורה):

    | שם המשתנה (Key)        | הערך (Value)                          |
    |------------------------|---------------------------------------|
    | `S3_BUCKET`            | `mytipul-uploads` (שם הדלי מסעיף 5)    |
    | `S3_REGION`            | `auto`                                |
    | `S3_ENDPOINT`          | ה-Endpoint מסעיף 14                    |
    | `S3_ACCESS_KEY_ID`     | ה-Access Key ID מסעיף 14              |
    | `S3_SECRET_ACCESS_KEY` | ה-Secret Access Key מסעיף 14          |

17. לחץ/י **Save Changes**. Render יבצע עדכון אוטומטי (כמה דקות).

## חלק 4 — בדיקה (נעשה יחד)

18. אחרי שהעדכון מסתיים: העלה/י מסמך חדש ← הורד/י אותו ← אמור לעבוד.
19. נוודא יחד שגם אחרי עדכון של האתר המסמך עדיין נפתח. מאותו רגע — הקבצים
    בטוחים לתמיד. 🎉

---

## חשוב לדעת

- 🔒 אל תשתף/י את ה-**Secret Access Key** עם אף אחד. אם דלף — מחק/י את הטוקן
  ב-Cloudflare וצור/י חדש.
- 📁 קבצים שהועלו **לפני** החיבור לא עוברים אוטומטית לענן (הם ממילא נמחקו
  מהדיסק הזמני). אם יש מסמך חשוב שהעלית לאחרונה — כדאי להעלות אותו מחדש אחרי
  שהחיבור פעיל.
- ⚙️ אם משאירים את המשתנים ריקים — המערכת ממשיכה לעבוד כמו היום (דיסק זמני).
  הכל מופעל רק כשכל 5 הערכים מוגדרים.

## למפתח (איך זה עובד בקוד)

- `src/lib/storage.ts` — `createStorage()` בוחר `S3StorageProvider` אם
  `S3_BUCKET` מוגדר, אחרת `LocalStorageProvider`.
- כתיבה מוסיפה `ServerSideEncryption: "AES256"`. אם R2 ידחה את ה-header הזה
  בעת הבדיקה — צריך להפוך אותו לאופציונלי ב-`S3StorageProvider.write()`.
- כל ה-callers של קבצי מטופלים כבר משתמשים ב-`storage` (ראה
  `PLAN-s3-migration.md` לסטטוס מלא).
