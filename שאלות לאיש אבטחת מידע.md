# שאלות וברורים לאיש אבטחת המידע

> מסמך זה מסכם את כל מה שצריך לשאול אותו ומה לבקש ממנו.
> אפשר להעתיק כל סעיף בנפרד ולשלוח לו בוואטסאפ/מייל.

---

## 📩 הודעה ראשונה לשלוח לו (העתק-הדבק):

```
שלום,

תודה על הבדיקה. כדי שאוכל לתקן את החורים בצורה מסודרת,
אני צריך ממך דוח כתוב עם הפרטים הבאים:

1. איך נכנסת בדיוק?
   - איזה URL / endpoint?
   - איזה parameter או שדה ניצלת?
   - איזו טכניקה? (SQL Injection / XSS / IDOR / Auth bypass / Brute force / משהו אחר?)

2. מה הצלחת לראות או לעשות אחרי שנכנסת?
   - נתונים של משתמשים אחרים?
   - הרשאות admin?
   - גישה לעסקאות / כרטיסי אשראי?
   - שינוי תוכן באתר?

3. רמת חומרה לפי הסיווג המקובל:
   Critical / High / Medium / Low

4. המלצות תיקון ספציפיות — אם יש לך.

5. האם בדקת רק את החור הזה, או סרקת את כל האתר?
   האם יש עוד חורים שגילית?

תודה רבה,
```

## 🔍 שאלות נוספות אם הוא לא נותן דוח כתוב:

```
1. האם תוכל לתת לי דוח כתוב? בלי דוח אני לא יכול לתקן בצורה מסודרת.

2. האם בדקת את הדברים הבאים:
   - סיסמאות חלשות / ברירת מחדל בחשבונות admin
   - NextAuth secret או משתני סביבה חשופים
   - API endpoints בלי authentication
   - IDOR (גישה לנתונים של משתמשים אחרים דרך שינוי ID)
   - SQL Injection
   - XSS בטפסים
   - CSRF על פעולות רגישות
   - חוסר ב-Rate Limiting (אפשרות לניחוש סיסמאות)
   - Webhook של Cardcom בלי validation של חתימה
   - secrets ב-git history
   - dependencies עם פרצות ידועות (CVEs)

3. האם הצלחת להגיע לנתוני מטופלים?

4. האם הצלחת לבצע עסקאות בלי תשלום אמיתי?

5. תוך כמה זמן הצלחת? (5 דקות זה מהיר מאוד —
   אומר שהיה חור בולט. מה הוא?)
```

## 💰 שאלות על המחיר והמשך השירות:

```
1. כמה זה עולה — בדיקה זו?

2. אם אני רוצה Pen Test מלא ומקיף — כמה זה עולה
   וכמה זמן זה לוקח?

3. אחרי שאתקן את החורים, האם תוכל לבדוק שוב
   ולוודא שהכל סגור? (Re-test)
   כמה זה עולה?

4. האם אתה ממליץ על בדיקה תקופתית? באיזה תדירות?
```

## ⚠️ דברים אדומים — אם הוא אומר אחד מאלה, **הוא לא מקצועי**:

- "פשוט תאמין לי שיש בעיה, אני לא נותן דוח"
- "אני לא יודע איך נכנסתי, סתם ניסיתי דברים"
- "תשלם לי ואני אגיד לך"
- מסרב לפרט מה הוא עשה
- אין לו הסמכה (CEH, OSCP, או דומה) ואין לו portfolio של לקוחות

איש אבטחה מקצועי **תמיד** נותן דוח כתוב מסודר.

---

## ✅ סימנים שהוא מקצועי:

- נותן דוח עם **CVSS score** (ציון חומרה תקני)
- מציין **CWE** (סיווג סוג החולשה)
- נותן **Proof of Concept** (הוכחה איך לשחזר)
- מציע **המלצות תיקון** ספציפיות
- מוכן לעשות **Re-test** אחרי התיקונים
- יש לו הסמכות: OSCP / CEH / CISSP / GPEN

## 🛡️ במקביל — סקירת אבטחה מלאה (מומלץ!)

גם אם איש האבטחה ייתן לי רק חור אחד, **כדאי שאעשה סקירה מלאה**
של כל הפרויקט. למה? כי אם נכנס ב-5 דקות, כנראה יש עוד חורים.

הסקירה תכלול:

- ✓ כל API endpoints (auth + authorization)
- ✓ ניהול סשנים ו-NextAuth
- ✓ אינטגרציית Cardcom + Webhooks
- ✓ Input validation בכל הטפסים
- ✓ Security Headers (CSP, HSTS וכו')
- ✓ Rate Limiting
- ✓ בדיקת secrets ב-git history
- ✓ בדיקת dependencies עם CVEs ידועים
- ✓ בדיקת הרשאות מסד נתונים
- ✓ CSP ו-CORS

**זמן משוער:** 30-60 דקות סריקה + 2-4 שעות תיקונים.
