# משימה: ייצוא דוח לרו"ח / דיווח עצמאי

## הקשר
המערכת היא מערכת ניהול מטופלים למטפלים (Next.js + Prisma + PostgreSQL).
יש דף קבלות (`receipts/page.tsx`) ודף תשלומים (`payments/page.tsx`).
כרגע כפתור "דוחות לרו"ח" נמצא בהיסטוריית תשלומים -- צריך להעביר אותו לדף הקבלות בגרסה משופרת.

## מה לעשות

### שלב 1: הסרת כפתור "דוחות לרו"ח" מהיסטוריית תשלומים
- **קובץ**: `src/app/(dashboard)/dashboard/payments/page.tsx`
- **שורות**: 721-791 (כפתור "דוחות לרו"ח" + dropdown שלו)
- **להסיר**: רק את כפתור "דוחות לרו"ח" ואת ה-dropdown שלו
- **לא לגעת**: בכפתור "ייצוא מפורט" (שורות 663-721) -- הוא נשאר!
- **לא לגעת**: בשום קוד אחר בדף התשלומים

### שלב 2: הוספת כפתור "דוח לרו"ח" בדף הקבלות
- **קובץ**: `src/app/(dashboard)/dashboard/receipts/page.tsx`
- **מיקום**: ליד כפתורי הייצוא הקיימים (CSV, הורדת PDF)
- **פורמט**: Excel (.xlsx) מרובה גליונות
- **בורר**: שנה (dropdown) + אופציה לרבעון ספציפי או שנה שלמה

### קבצים רלוונטיים
- `src/app/(dashboard)/dashboard/receipts/page.tsx` -- דף הקבלות (הוספת כפתור)
- `src/app/(dashboard)/dashboard/payments/page.tsx` -- דף תשלומים (הסרת כפתור)
- `src/lib/export-utils.ts` -- פונקציות ייצוא קיימות (יש שם `exportAccountantExcel`)
- `src/app/api/payments/route.ts` -- API של תשלומים

### ספריות קיימות בפרויקט
- `xlsx` -- ליצירת Excel
- `jspdf` + `jspdf-autotable` -- ליצירת PDF
- `date-fns` -- פורמט תאריכים

### API קיים
- `GET /api/payments` -- מחזיר את כל התשלומים כולל `hasReceipt`, `receiptNumber`, `receiptUrl`
- `GET /api/user/profile` -- מחזיר פרופיל (שם עסק וכו')
